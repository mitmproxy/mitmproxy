Step 4: Replay Requests

Another powerful feature of mitmproxy is replaying previous flows. Two types of replays are supported:

  • Client-side Replay: mitmproxy replays previous client requests, i.e., sends them to the server again, and the server replies with the reponse.
  • Server-side Replay: mitmproxy replays server responses for requests that match an earlier recorded request.

In this tutorial we focus on the more common use case of client-side replays. See the docs for more info on server-side replay.
